📘 4주차 알고리즘개론 대면수업 문제

✅ O/X 문제
	1.	배열의 최댓값을 구하는 알고리즘에서 비교 연산 횟수는 항상 배열 상태와 무관하게 동일하다. (O / X)
	2.	선형 검색에서 평균 비교 횟수는 데이터가 정렬되어 있을 경우 줄어든다. (O / X)
	3.	보초법을 사용하면 시간복잡도가 O(n)에서 O(log n)으로 감소한다. (O / X)
	4.	이진 검색에서 매 반복마다 검색 범위는 항상 정확히 절반으로 줄어든다. (O / X)
	5.	이진 검색은 삽입/삭제가 빈번한 경우 비효율적일 수 있다. (O / X)
	6.	리스트는 서로 다른 자료형을 저장할 수 있다. (O / X)
	7.	기수 변환에서 역순 출력은 상위 자리수부터 계산되기 때문이다. (O / X)
	8.	소수 판별 알고리즘에서 √n까지만 나눗셈을 수행해도 되는 이유는 약수가 항상 쌍으로 존재하기 때문이다. (O / X)
	9.	해시 테이블에서 충돌이 발생하더라도 평균 시간복잡도는 여전히 O(1)을 유지할 수 있다. (O / X)
	10.	선형 검색은 배열이 정렬되어 있는 경우에도 항상 이진 검색보다 비효율적이다. (O / X)
	11.	이진 검색에서 중앙 인덱스 계산 시 (pl + pr) // 2를 사용하는 이유는 정수 인덱스를 유지하기 위함이다. (O / X)
	12.	이진 검색에서 pl > pr이 되는 순간 검색은 실패로 종료된다. (O / X)
	13.	배열을 역순으로 정렬하는 알고리즘에서 원소 수가 n일 때 교환 횟수는 항상 n번이다. (O / X)
	14.	리스트 슬라이스 s[::-1]은 원본 리스트를 변경하지 않고 새로운 리스트를 생성한다. (O / X)
	15.	해시 함수는 서로 다른 키에 대해 항상 서로 다른 해시값을 반환해야 한다. (O / X)
	16.	체인법 해시에서 모든 데이터가 하나의 버킷에 저장되면 검색 시간복잡도는 O(n)이 된다. (O / X)
	17.	선형 검색에서 최악의 경우 비교 횟수는 배열의 원소 개수와 같다. (O / X)
	18.	이진 검색의 시간복잡도는 항상 정확히 log₂n번의 비교로 결정된다. (O / X)
	19.	정렬된 배열에서 새로운 원소를 삽입할 때는 최악의 경우 O(n)의 시간이 필요하다. (O / X)
	20.	해시 테이블은 평균적으로 빠르지만, 최악의 경우 선형 검색과 동일한 성능이 될 수 있다. (O / X)

def bin_search(a, key):
    pl = 0
    pr = len(a) - 1
    while pl <= pr:
        pc = (pl + pr) // 2
        if a[pc] == key:
            return pc
        elif a[pc] < key:
            pl = pc + 1
        else:
            pr = pc - 1
    return -1

a = [3, 8, 15, 21, 27, 32, 40]
print(bin_search(a, 27))

① 3
② 4
③ 5
④ -1

def hash_func(key):
    return key % 5

table = [None] * 5

def insert(key):
    h = hash_func(key)
    while table[h] is not None:
        h = (h + 1) % 5
    table[h] = key

    insert(k)
