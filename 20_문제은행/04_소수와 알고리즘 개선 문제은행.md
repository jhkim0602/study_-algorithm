## 개념노트용 핵심 문제

<details>
<summary>문제 8. 소수 판별 알고리즘에서 <code>sqrt(n)</code>까지만 나눗셈을 수행해도 되는 이유는 약수가 항상 쌍으로 존재하기 때문이다. (O / X)</summary>

출처: `IMG_8868`, 8번

<details>
<summary>해설 보기</summary>

- 정답: O
- 이유: 어떤 수 `n`의 약수는 `(a, b)`처럼 쌍으로 존재하고, 둘 다 `sqrt(n)`보다 클 수는 없다. 따라서 `sqrt(n)` 이하에서 약수를 찾지 못하면 그 이후도 볼 필요가 없다.

</details>
</details>

<details>
<summary>문제 43. 다음 코드의 출력 결과는?</summary>

출처: `IMG_8872`, 43번

```python
count = 0
for n in range(2, 8):
    for i in range(2, n):
        count += 1
        if n % i == 0:
            break
    else:
        print(n, end=' ')
```

1. `2 3 5 7`
2. `2 3 4 5 6 7`
3. `3 5 7`
4. `2 5 7`

<details>
<summary>해설 보기</summary>

- 정답: 1번
- 이유: `for-else`의 `else`는 내부 `break` 없이 반복문이 끝났을 때만 실행된다. 따라서 약수를 만나지 않은 수, 즉 소수만 출력된다.

</details>
</details>

<details>
<summary>문제 44. 다음 소수 판정 개선 코드의 핵심 아이디어는?</summary>

출처: `IMG_8872`, 44번

```python
while i < ptr and prime[i] * prime[i] <= n:
```

1. 배열을 뒤에서부터 탐색한다
2. 이미 구한 소수만 이용하고, `sqrt(n)` 이하까지만 검사한다
3. 모든 정수로 나누어 본다
4. 해시 함수를 사용한다

<details>
<summary>해설 보기</summary>

- 정답: 2번
- 이유: 이미 구한 소수 목록을 재사용하고, `prime[i] * prime[i] <= n` 조건으로 필요한 범위까지만 검사해 중복 계산을 줄인다.

</details>
</details>

## 추가 문제

<details>
<summary>문제 45. 다음 코드의 시간복잡도로 가장 적절한 것은?</summary>

출처: `IMG_8872`, 45번

```python
for n in range(2, N):
    for i in range(2, n):
        if n % i == 0:
            break
```

1. `O(log n)`
2. `O(n)`
3. `O(n log n)`
4. `O(n^2)`

<details>
<summary>해설 보기</summary>

- 정답: 4번
- 이유: 가장 단순한 상한으로 보면 바깥 반복과 안쪽 반복이 함께 커지므로 이중 반복의 `O(n^2)`로 보는 것이 적절하다.

</details>
</details>
