# 05 - 비교표

## 배열, 리스트, 튜플

| 항목 | 배열 | 리스트 | 튜플 |
| --- | --- | --- | --- |
| 보는 관점 | 여러 값을 한 덩어리로 다루는 개념 | 파이썬의 변경 가능한 시퀀스 | 파이썬의 변경 불가능한 시퀀스 |
| 핵심 장점 | 문제를 반복과 위치 접근으로 바꾸기 쉬움 | 실습과 수정이 편함 | 변경되면 안 되는 구조 표현에 적합 |
| 자주 하는 오해 | 단순 저장소로만 봄 | 배열과 완전히 다른 개념이라고 느낌 | 괄호 모양만 다른 리스트라고 느낌 |

## 선형 검색, 보초법, 이진 검색

| 항목 | 선형 검색 | 보초법 | 이진 검색 |
| --- | --- | --- | --- |
| 전제 | 없음 | 없음 | 정렬 필요 |
| 핵심 아이디어 | 앞에서부터 하나씩 확인 | 끝 검사 구조 단순화 | 절반씩 버리기 |
| 시간 감각 | 데이터가 크면 느려짐 | 본질은 선형 검색과 같음 | 큰 입력에서 매우 강함 |
| 구현 난도 | 가장 쉬움 | 쉬움 | 경계 관리가 필요 |
| 자주 하는 실수 | 실패 반환값 누락 | 원본 배열 훼손 | 정렬 전제 무시 |

## 단순 소수 검사와 개선 소수 검사

| 항목 | 단순 버전 | 개선 버전 |
| --- | --- | --- |
| 검사 대상 | 거의 모든 수 | 필요한 후보만 |
| 재사용 | 없음 | 이미 찾은 소수 재사용 |
| 종료 범위 | 끝까지 | 제곱근까지 |
| 장점 | 이해가 쉬움 | 훨씬 적은 연산 |
| 비용 | 느림 | 소수 배열 저장 필요 |

## 정렬된 배열과 해시

| 항목 | 정렬된 배열 | 해시 |
| --- | --- | --- |
| 검색 | 빠를 수 있음 | 평균적으로 매우 빠름 |
| 삽입 | 중간 삽입이 불편함 | 버킷에 따라 비교적 수월 |
| 핵심 전략 | 순서 유지 | 위치 계산 |
| 약점 | 삽입/삭제 이동 비용 | 충돌 |
| 이해 포인트 | 정렬 유지의 비용 | 충돌 처리가 설계의 핵심 |

## 시간 복잡도와 공간 복잡도

| 항목 | 시간 복잡도 | 공간 복잡도 |
| --- | --- | --- |
| 보는 것 | 연산량 증가 | 추가 메모리 증가 |
| 질문 | 입력이 커질수록 얼마나 느려지나 | 입력 외에 메모리를 얼마나 더 쓰나 |
| 예시 | 선형 검색 `O(n)`, 이진 검색 `O(log n)` | 복사본 생성, 해시 버킷 추가 |
| 자주 하는 오해 | 실행 시간과 완전히 같다고 생각 | 시간보다 덜 중요하다고 무시 |
