# 06 - 미니 문제 세트

이 노트는 손으로 풀어보는 짧은 문제 모음이다.  
먼저 스스로 답하고, 그 다음 아래의 해설 뼈대를 확인한다.

## 1. 배열 입문

### 문제

`score1`, `score2`, `score3`, `score4`로 점수를 관리할 때의 불편을 2가지 적어라.

### 해설 뼈대

- 여러 값을 반복문으로 다루기 어렵다.
- 최대값, 합계, 평균 계산 때 같은 코드를 반복하게 된다.

## 2. 슬라이싱

### 문제

`values = [10, 20, 30, 40, 50, 60]`일 때:

- `values[:3]`
- `values[3:]`
- `values[1:5:2]`

의 결과를 적어라.

### 해설 뼈대

- 끝 인덱스는 포함되지 않는다.
- step이 2이면 한 칸씩 건너뛴다.

## 3. 최대값 찾기

### 문제

왜 `maximum = 0`은 안전한 초기값이 아닌가?

### 해설 뼈대

- 배열이 모두 음수일 수 있다.
- 실제 데이터 안의 값으로 시작해야 가짜 후보가 끼어들지 않는다.

## 4. 진수 변환

### 문제

왜 진수 변환에서는 나머지를 모은 뒤 마지막에 뒤집어야 하는가?

### 해설 뼈대

- 먼저 나온 나머지는 낮은 자리수다.
- 최종 문자열은 높은 자리수부터 보여야 한다.

## 5. 알고리즘 개선

### 문제

왜 `p * p <= n`까지만 검사해도 되는가?

### 해설 뼈대

- 약수는 쌍으로 나타난다.
- 제곱근보다 큰 약수가 있다면, 대응되는 더 작은 약수가 이미 앞에서 발견됐어야 한다.

## 6. 검색

### 문제

정렬되지 않은 배열에 이진 검색을 쓰면 왜 안 되는가?

### 해설 뼈대

- 이진 검색은 가운데 값을 보고 절반을 버리는 알고리즘이다.
- 정렬이 없으면 버린 절반 안에 key가 없다고 보장할 수 없다.

## 7. 복잡도

### 문제

왜 `O(2n + 3)`을 보통 `O(n)`이라고 쓰는가?

### 해설 뼈대

- 입력이 커질수록 상수항 영향은 작아진다.
- 가장 크게 자라는 항이 전체 성능을 지배한다.

## 8. 해시

### 문제

충돌이 생겼다고 해서 왜 해시 테이블이 실패한 것은 아닌가?

### 해설 뼈대

- 버킷 수가 유한하므로 충돌은 자연스럽다.
- 중요한 것은 충돌을 처리하는 방식이며, 체인법은 연결 리스트로 처리한다.

## 9. 말로 설명 연습

### 문제

아래 문장을 각각 3문장으로 설명해 보라.

- 배열은 왜 필요한가?
- 이진 검색은 왜 빠른가?
- 해시는 왜 충돌을 허용하는가?

### 해설 뼈대

- 정의
- 필요성
- 핵심 동작 원리
