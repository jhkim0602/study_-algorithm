# 02 - 질문 은행

이 노트는 단순 암기 확인용이 아니라, 말로 설명하는 훈련용이다.  
각 질문에 답할 때는 가능하면 아래 순서를 지킨다.

1. 정의를 한 문장으로 말한다.
2. 왜 필요한지 말한다.
3. 짧은 예시를 든다.
4. 자주 하는 실수나 전제를 덧붙인다.

## 배열과 리스트

- 배열이 왜 필요한가?
  답할 때 포함할 것: 변수 여러 개를 따로 관리할 때의 불편함, 여러 값을 한 덩어리로 보는 장점
- 리스트와 튜플의 차이는 무엇인가?
  답할 때 포함할 것: 변경 가능성, 표현 의도, 예시 데이터
- 슬라이싱에서 끝 인덱스가 포함되지 않는 이유를 예시로 설명할 수 있는가?
  답할 때 포함할 것: 구간 분할, 경계선 감각, `a[:3]`와 `a[3:]`
- 음수 인덱스는 왜 편리한가?
  답할 때 포함할 것: 마지막 원소 접근, 길이 계산 없이 끝부분 다루기

## 배열 순회와 모듈화

- 최대값 찾기에서 왜 첫 원소를 초기값으로 두는가?
  답할 때 포함할 것: 음수 배열 반례, 실제 후보만 답으로 삼아야 한다는 점
- 스캔 패턴이란 무엇인가?
  답할 때 포함할 것: 후보 설정, 순회, 갱신
- 왜 `End` 같은 종료 신호가 유용한가?
  답할 때 포함할 것: 반복 횟수를 미리 모를 때, 종료 사건 중심 반복
- 함수로 분리하면 무엇이 좋아지는가?
  답할 때 포함할 것: 재사용, 테스트, 입력/계산 분리
- 좋은 주석은 무엇을 써야 하는가?
  답할 때 포함할 것: 무엇보다 왜를 남긴다

## 배열 응용과 진수 변환

- 배열 뒤집기에서 왜 절반만 반복하는가?
  답할 때 포함할 것: 한 쌍은 한 번만 바꾸면 충분하다는 점
- 진수 변환이 왜 배열/문자열 단원에 들어오는가?
  답할 때 포함할 것: 나머지 저장, 역순 처리, 기호표 사용
- 진수 변환에서 나머지를 모은 뒤 뒤집는 이유는 무엇인가?
  답할 때 포함할 것: 먼저 나온 나머지가 낮은 자리수라는 점
- `DIGITS` 문자열은 어떤 역할을 하는가?
  답할 때 포함할 것: 숫자를 문자 기호로 번역하는 표

## 알고리즘 개선

- 짝수를 건너뛰면 왜 빨라지는가?
  답할 때 포함할 것: 2를 제외한 짝수는 소수가 아니라는 점
- 왜 이미 찾은 소수들만 다시 검사하면 되는가?
  답할 때 포함할 것: 합성수의 약수는 결국 소수들의 곱으로 표현된다는 점
- 왜 `p * p <= n`까지만 검사해도 되는가?
  답할 때 포함할 것: 약수 쌍, 제곱근 이후 검사의 중복성
- 개선 알고리즘은 무엇을 저장하고 무엇을 아끼는가?
  답할 때 포함할 것: 소수 배열 저장, 나눗셈 횟수 절약

## 검색과 복잡도

- 키는 무엇인가?
  답할 때 포함할 것: 검색 기준, 같은 데이터도 키 선택에 따라 전략이 달라진다는 점
- 선형 검색과 이진 검색의 차이는 무엇인가?
  답할 때 포함할 것: 전제 조건, 탐색 방식, 비교 횟수
- 보초법은 무엇을 단순화하는가?
  답할 때 포함할 것: 반복문의 종료 검사 구조
- 왜 `O(n)`보다 `O(log n)`이 더 빠른가?
  답할 때 포함할 것: 절반씩 버리는 구조, 큰 입력에서의 차이
- 정렬 비용까지 함께 봐야 한다는 말은 무슨 뜻인가?
  답할 때 포함할 것: 한 번 찾는 상황과 여러 번 찾는 상황의 차이

## 해시

- 해시는 왜 빠른가?
  답할 때 포함할 것: 비교 반복보다 위치 계산, 후보 구간 축소
- 충돌은 왜 생기는가?
  답할 때 포함할 것: 버킷 수의 한계, 서로 다른 키의 동일 버킷 도착
- 체인법은 충돌을 어떻게 처리하는가?
  답할 때 포함할 것: 같은 버킷 원소를 연결 리스트로 저장
- 체인법에서 삭제 시 이전 노드를 왜 기억해야 하는가?
  답할 때 포함할 것: 연결을 다시 이어 주어야 한다는 점
- 해시가 평균적으로 빠르지만 최악에는 느려질 수 있는 이유는 무엇인가?
  답할 때 포함할 것: 충돌 집중, 긴 체인, 버킷 수와 분산 문제
