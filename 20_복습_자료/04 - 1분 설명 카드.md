# 04 - 1분 설명 카드

이 노트는 "짧고 선명하게 설명하는 연습"용이다.  
각 주제마다 1분 안에 아래 순서로 말해 본다.

1. 무엇인가
2. 왜 필요한가
3. 어떻게 동작하는가
4. 어디서 자주 틀리는가

## 배열

- 무엇인가: 여러 값을 한 덩어리로 다루는 구조다.
- 왜 필요한가: 변수 여러 개를 따로 관리하는 불편을 줄이고, 반복과 위치 접근으로 문제를 단순화한다.
- 어떻게 동작하는가: 인덱스로 한 칸을 보고, 슬라이싱으로 구간을 본다.
- 자주 틀리는가: 슬라이싱 끝 인덱스를 포함된다고 착각한다.

## 최대값 찾기

- 무엇인가: 배열을 한 번 훑으며 가장 큰 값을 찾는 스캔 문제다.
- 왜 필요한가: 순회와 후보 갱신이라는 공통 패턴을 가장 단순하게 보여 준다.
- 어떻게 동작하는가: 첫 원소를 후보로 두고, 더 큰 값을 만날 때마다 갱신한다.
- 자주 틀리는가: `0` 같은 임의 초기값을 두어 음수 배열에서 틀린다.

## 모듈화

- 무엇인가: 계산 로직과 입출력 로직을 분리하는 설계 방식이다.
- 왜 필요한가: 재사용, 테스트, 읽기 쉬움을 확보하기 위해서다.
- 어떻게 동작하는가: 함수는 계산만 맡고, 입력과 출력은 바깥 코드가 맡는다.
- 자주 틀리는가: 함수 안에 입력까지 넣어 재사용성을 떨어뜨린다.

## 배열 뒤집기

- 무엇인가: 양끝 원소를 교환하며 배열 순서를 반대로 만드는 작업이다.
- 왜 필요한가: 위치 교환과 대칭 구조를 이해하는 대표 예제다.
- 어떻게 동작하는가: 앞쪽 인덱스와 대응되는 뒤쪽 인덱스를 바꾸고 중앙에서 멈춘다.
- 자주 틀리는가: 끝까지 반복해 같은 쌍을 다시 바꾼다.

## 진수 변환

- 무엇인가: 어떤 수를 다른 진법 문자열로 바꾸는 알고리즘이다.
- 왜 필요한가: 나머지, 역순, 문자열 조립이 함께 등장하는 좋은 응용 문제다.
- 어떻게 동작하는가: base로 나눈 나머지를 계속 쌓고 마지막에 뒤집는다.
- 자주 틀리는가: 나머지가 쌓인 순서가 곧 정답이라고 착각한다.

## 알고리즘 개선

- 무엇인가: 같은 정답을 더 적은 연산으로 구하는 방식이다.
- 왜 필요한가: 단순 구현에서 효율적인 설계로 넘어가는 첫 단계이기 때문이다.
- 어떻게 동작하는가: 짝수를 건너뛰고, 이미 찾은 소수를 재사용하고, 제곱근까지만 검사한다.
- 자주 틀리는가: 왜 빨라졌는지 설명하지 못하고 코드만 외운다.

## 선형 검색

- 무엇인가: 앞에서부터 차례대로 key를 찾는 검색이다.
- 왜 필요한가: 가장 단순하고 전제가 거의 없는 기본 검색이기 때문이다.
- 어떻게 동작하는가: 배열 처음부터 끝까지 같은지 비교한다.
- 자주 틀리는가: 실패 시 무엇을 반환할지 미리 정하지 않는다.

## 이진 검색

- 무엇인가: 정렬된 배열에서 절반씩 버리며 찾는 검색이다.
- 왜 필요한가: 입력이 커질수록 비교 횟수를 크게 줄일 수 있다.
- 어떻게 동작하는가: 가운데 값을 보고 왼쪽 또는 오른쪽 절반을 버린다.
- 자주 틀리는가: 정렬되지 않은 배열에 적용한다.

## 복잡도

- 무엇인가: 입력이 커질수록 연산량과 메모리 사용이 어떻게 늘어나는지 보는 기준이다.
- 왜 필요한가: 돌아가는 코드와 확장 가능한 코드를 구분하기 위해서다.
- 어떻게 동작하는가: 정확한 시간보다 성장률을 본다.
- 자주 틀리는가: 한 번의 실행 시간만 보고 성능을 단정한다.

## 해시

- 무엇인가: 키로부터 저장 위치를 계산해 찾는 자료구조 방식이다.
- 왜 필요한가: 비교를 반복하지 않고 빠르게 후보 구간을 좁히기 위해서다.
- 어떻게 동작하는가: 해시 함수로 버킷을 정하고, 체인법이면 그 안에서 연결 리스트를 본다.
- 자주 틀리는가: 충돌이 생기면 구조가 실패했다고 생각한다.
