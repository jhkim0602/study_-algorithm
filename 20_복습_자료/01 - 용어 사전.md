# 01 - 용어 사전

## 배열

- 뜻: 같은 종류의 데이터를 한 덩어리로 다루는 구조
- 꼭 알아야 할 점: 배열은 값을 모으는 상자이면서, 문제를 반복과 위치 접근으로 바꾸는 도구다.
- 자주 하는 오해: 배열을 단순 저장소로만 보고, 문제 구조를 정리해 주는 역할을 놓친다.

## 리스트

- 뜻: 변경 가능한 파이썬 시퀀스
- 꼭 알아야 할 점: 값을 추가, 수정, 삭제할 수 있어 실습과 알고리즘 입문에 가장 많이 쓴다.
- 자주 하는 오해: 배열과 리스트를 완전히 다른 개념이라고 느끼거나, 반대로 아무 차이도 없다고 느낀다.

## 튜플

- 뜻: 변경 불가능한 파이썬 시퀀스
- 꼭 알아야 할 점: 바뀌면 안 되는 묶음을 표현할 때 적합하다.
- 자주 하는 오해: 괄호 모양만 다른 리스트라고 생각한다.

## 인덱스

- 뜻: 한 원소의 위치 번호
- 꼭 알아야 할 점: 인덱스는 값의 이름 대신 위치로 접근하게 해 준다.
- 자주 하는 오해: 첫 번째 원소를 `1`로 접근하려고 한다.

## 슬라이싱

- 뜻: 구간 단위 추출
- 꼭 알아야 할 점: 끝 인덱스를 포함하지 않기 때문에 구간이 자연스럽게 이어지고 쪼개진다.
- 자주 하는 오해: `a[2:5]`에서 5번째 원소도 포함된다고 생각한다.

## 스캔 패턴

- 뜻: 배열을 앞에서부터 끝까지 훑으며 답 후보를 갱신하는 방식
- 꼭 알아야 할 점: 최대값, 최소값, 개수 세기, 조건 만족 여부 확인에 공통으로 쓰인다.
- 자주 하는 오해: 문제마다 완전히 새로운 방식이 필요하다고 느낀다.

## 키

- 뜻: 검색 기준이 되는 값
- 꼭 알아야 할 점: 같은 데이터도 어떤 키를 고르느냐에 따라 검색 전략이 달라진다.
- 자주 하는 오해: 데이터 전체와 키를 구분하지 못한다.

## 선형 검색

- 뜻: 앞에서부터 차례대로 찾는 검색
- 꼭 알아야 할 점: 정렬이 필요 없고 가장 범용적이다.
- 자주 하는 오해: 느리다는 이유만으로 항상 나쁜 알고리즘이라고 생각한다.

## 보초법

- 뜻: 끝에 찾는 값을 붙여 비교 구조를 단순화한 선형 검색
- 꼭 알아야 할 점: 핵심 아이디어는 그대로 두고 반복문의 조건만 단순하게 만든다.
- 자주 하는 오해: 완전히 다른 검색 알고리즘이라고 생각한다.

## 이진 검색

- 뜻: 정렬된 배열에서 가운데 기준으로 범위를 반씩 줄이는 검색
- 꼭 알아야 할 점: 빠르지만 정렬이라는 전제가 반드시 필요하다.
- 자주 하는 오해: 정렬되지 않은 데이터에도 적용하려 한다.

## 시간 복잡도

- 뜻: 입력 크기에 따라 계산량이 어떻게 늘어나는지 보는 기준
- 꼭 알아야 할 점: 정확한 초 단위 시간보다 성장 패턴을 본다.
- 자주 하는 오해: 한 번 재 본 실행 시간만으로 복잡도를 판단한다.

## 공간 복잡도

- 뜻: 추가로 필요한 메모리 양을 보는 기준
- 꼭 알아야 할 점: 더 빠른 알고리즘이 더 많은 메모리를 요구할 수도 있다.
- 자주 하는 오해: 시간보다 덜 중요하다고 생각해 무시한다.

## 해시 함수

- 뜻: 키를 인덱스로 바꾸는 함수
- 꼭 알아야 할 점: 좋은 해시는 키를 여러 버킷에 고르게 퍼뜨리도록 돕는다.
- 자주 하는 오해: 충돌이 하나도 없어야 좋은 함수라고 생각한다.

## 충돌

- 뜻: 서로 다른 키가 같은 버킷으로 가는 현상
- 꼭 알아야 할 점: 충돌은 피할 수 없는 현상이며, 중요한 것은 충돌 처리 방식이다.
- 자주 하는 오해: 충돌이 생기면 자료구조가 틀렸다고 생각한다.

## 체인법

- 뜻: 충돌한 원소를 연결 리스트로 저장하는 방식
- 꼭 알아야 할 점: 버킷까지는 빠르게 가고, 버킷 안에서는 연결 리스트를 선형 탐색한다.
- 자주 하는 오해: 해시를 쓰면 모든 탐색이 무조건 한 번에 끝난다고 생각한다.
